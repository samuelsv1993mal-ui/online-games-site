# SMQ Games — офлайн по Wi‑Fi / точке доступа

Этот режим нужен, когда нет интернета, но несколько телефонов должны играть вместе.
Один Android-телефон запускает сайт как локальный сервер через Termux. Остальные телефоны подключаются к его точке доступа и открывают локальный адрес.

## Что нужно

1. Android-телефон ведущего.
2. Termux на телефоне ведущего.
3. Node.js в Termux.
4. Проект SMQ Games на телефоне.
5. Один раз заранее выполнить `npm install`, пока интернет есть. После этого играть можно без интернета.
6. Другие телефоны должны быть подключены к точке доступа ведущего.

## Первый запуск, пока есть интернет

```bash
pkg update -y
pkg install nodejs git unzip -y
cd ~/online-games-site
npm install
```

## Запуск без интернета

1. Включи на телефоне точку доступа.
2. Подключи телефоны друзей к этой точке доступа.
3. В Termux запусти:

```bash
cd ~/online-games-site
bash scripts/start-wifi-server.sh
```

Скрипт покажет адреса вида:

```text
http://192.168.43.1:3000
http://192.168.1.25:3000
```

Друзья открывают этот адрес в браузере. После этого комнаты, ссылки игрока и ссылки зрителя работают внутри локальной Wi‑Fi сети.

## Важно

- Render в этом режиме не нужен.
- Интернет не нужен после запуска локального сервера.
- Google-вход без интернета может не работать, поэтому лучше использовать гостевой вход.
- Если телефон блокирует Termux в фоне, держи Termux открытым во время игры.
- Если адрес не открывается, выключи и снова включи точку доступа, потом перезапусти сервер.

## Новый телефонный клиент

В версии v23 добавлены два скрипта:

```bash
bash scripts/install-phone-client.sh
bash scripts/start-phone-client.sh
```

Первый скрипт готовит телефон и создаёт ярлык для Termux:Widget. Второй запускает телефон как локальный Wi‑Fi сервер и показывает адрес/QR для друзей.
